# 版本规划

## 版本号策略
采用 **语义化版本** (SemVer): `主版本.次版本.补丁`

- **主版本**: 大架构变更、破坏性 API 更改
- **次版本**: 新功能、非破坏性增强
- **补丁**: Bug 修复、性能优化、微小改进

## 版本控制规范

### Git 分支模型
```
main         ← 稳定发布
  ├─ develop    ← 开发主分支
  │    ├─ feat/*    ← 功能分支
  │    ├─ fix/*     ← 修复分支
  │    └─ refactor/* ← 重构分支
  └─ release/*  ← 发布准备分支
```

### 提交信息规范 (Conventional Commits)
```
feat: 新功能
fix: Bug 修复
docs: 文档变更
style: 样式/格式变更
refactor: 重构
perf: 性能优化
chore: 工具/构建/依赖
```

### 示例
```
feat(editor): 添加 CodeMirror 搜索高亮功能
fix(preview): 修复 KaTeX 公式渲染溢出问题
docs: 更新 README 安装说明
chore: 升级 Electron 到 v33
```

### 发布流程
1. `develop` → `release/v0.x.0`
2. 测试、修复 Bug
3. `release/v0.x.0` → `main` + Tag `v0.x.0`
4. `main` 触发 CI 自动构建 + 发布
